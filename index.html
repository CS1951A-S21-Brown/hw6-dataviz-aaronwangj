List 3 reasons why D3 was helpful and improved your visualization (6 points)
1. D3 is extremely flexible in visual styling, which allows me to customize the graphs the way I want them. For example, I can easily change the font, color, backgrounds, etc. of the graph.
2. D3 was helpful in that there was a lot of online documentation regarding what I needed to do to get the visualization the way I wanted it to. Furthermore, because it is widely used, there are a lot of great examples to look at and learn from.
3. D3 is extensible in that it isn't data dependent; that is, I can make a visualization for one data set, and then input another data set of the same format and the visualization would work on this new data set. This is great for making a lot of similar visualizations on different data.

List 3 reasons why D3 would not be the best tool for creating a visualization (6 points)
1. D3 would not be the best tool for creating a visualization if one does not understand JS, HTML, or CSS. For example, I had to spend a lot of time learning JS before I could understand how to use D3 to my advantage.
2. D3 would not be the best tool for creating a visualization if you only need a simple graph without much customization. Programs like Microsoft Excel already provide intuitive GUIs for cookie-cutter graphs that may be enough for some people.
3. D3 would not be the best tool for creating a visualization if the visualization is not meant to be done on a webpage. A lot of other frameworks such as ggplot in R and Matplotlib in Python provide arguably better support for data visualizations in realms outside of web development, such as in machine learning, computer vision, etc.

Evaluate the accessibility of your dashboard based on the readings in the “Before you begin” section. What kinds of users might find this dashboard accessible and who might have more difficulty? What additional actions might you take to make this dashboard more accessible to all audiences? Your response should refer to at least one of the readings and be about 1 paragraph.
Those who are not visually-impaired may find this dashboard much more accessible than those who are visually-impaired. Thus, actions should be taken to ensure that this dashboard is more accessible for these people. From “A Comprehensive Guide to Accessible Data Visualization," I learned that many visually-impaired people rely on screen readers. Hence, I should write a brief summary of the data visualization's finding under each plot in order for the screen reader to read this to the user. My dashboard may also give greater difficulty to those who are color-blind, and thus may not completely understand the visualizations I've created. In this case, as noted in “Why Accessibility Is at the Heart of Data Visualization,” I should include a link to the csv file of the data in order for the user to take a better look at the data themselves.
